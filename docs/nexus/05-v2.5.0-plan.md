## Thesis

Nexus's competitive problem is **not** missing Cypher features (~85% parity
already) — it is that **correctness depends on the transport** (five write
implementations), **concurrency is gated by a global engine lock**, and the
**benchmark story is unpublishable** (contradictory numbers, no KNN recall
curves). 2.5.0 fixes the trust layer: one write path, lock-free reads, a
re-baselined public benchmark, plus the cheap compatibility tail. Bolt and
columnar execution are the 2.6 epics.

## Goals (verifiable)

| Goal | Success criterion |
|---|---|
| G1 One write path | `write_ops.rs` deleted; parity harness green; same battery gives identical results over HTTP/RPC/RESP3/GraphQL/streaming/embedded |
| G2 Zero known silent-loss bugs | B1–B8 from [02](02-bug-inventory.md) each have a red-then-green test |
| G3 Lock-free reads | MATCH-only queries bypass `engine.write()`; concurrency bench shows multi-core scaling |
| G4 Headline perf gaps closed | COUNT(*) ~O(1); rel traversal gap cut ≥30%; join ordering statistics-driven |
| G5 Trustworthy benchmarks | One re-baselined vs-Neo4j report; CREATE-rel contradiction resolved; KNN recall@k published |
| G6 Compat tail trimmed | randomUUID + string/math/list fns + elementId format shipped; compat suite extended to run per-transport |

## Explicitly deferred to 2.6+ (tracked, not forgotten)

- **Columnar/vectorized execution representation** (perf #2 — L, foundational;
  requires its own ADR + design doc)
- **Page-cache 2Q/W-TinyLFU** (perf #6)
- **Group-commit WAL default** (perf #7 second half)
- **Executor-native MERGE/SET/REMOVE/FOREACH operators** (unification Step 8 —
  own ADR; hard to reverse)
- **APOC top-20 procedures**, streaming cursors, UDF bodies, relationship
  existence constraints, `CALL {} IN TX` concurrency (needs V2 sharding)

## Risk register (top 5)

1. **Behavioral divergences surfaced by the parity harness** may change
   client-visible HTTP outputs → resolve in favor of Neo4j semantics,
   document in CHANGELOG "Fixed", update compat suite.
2. **Audit-log relocation** must land before `write_ops.rs` deletion
   (compliance regression otherwise) — blocking checklist item.
3. **Lock-free read routing** risks read-your-own-write anomalies within a
   session → keep writes + in-transaction reads on the engine; only
   autocommit pattern-reads go lock-free; add snapshot-visibility tests.
4. **Benchmark re-baseline could reveal regressions** (CREATE-rel truth
   unknown) → treat as measurement, not failure; fix-forward items feed 2.6.
5. **Scope creep from Bolt** → phase8 is last and cuttable; 2.5.0 ships
   without it if needed.
