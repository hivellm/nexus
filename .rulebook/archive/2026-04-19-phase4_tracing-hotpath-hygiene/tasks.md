## 1. Implementation

- [x] 1.1 Enumerated every `tracing::info!` / `debug!` call in `nexus-core/src/executor/`. Starting point: 12 `info!` calls (6 context.rs, 3 mod.rs, 2 scan.rs, 1 join.rs) and 123 `debug!` calls (densest clusters: union.rs 19, path.rs 19, expand.rs 18, mod.rs 18, project.rs 13). Every `info!` call fired per-query under the default `RUST_LOG=info`. The per-row debug! clusters in expand / path / union / project / create / mod were catalogued as hot-path; the per-operator debug! in filter / planner / dispatch / eval stayed at `debug!` (once-per-query level, useful during targeted debugging without flooding logs).
- [x] 1.2 All 12 `info!` calls downgraded. The six ADVANCED JOIN info lines in `executor/context.rs`, the three cache-hit / direct-exec lines in `executor/mod.rs`, and the one optimized-join line in `executor/operators/join.rs` collapsed to `trace!` — they were pure flavor text on every query. The two `scan.rs` `"INDEX OPPORTUNITY"` lines consolidated into a single structured `tracing::debug!(prop, accesses, "index opportunity: …")` event (not `trace!` because they're actionable tuning hints that callers actively want when investigating slow WHERE clauses). Per-row `debug!` clusters in create / expand / mod / path / project / union collapsed to `trace!` via a one-file-per-pass `replace_all`. `filter.rs`, `join.rs`, `planner/queries.rs`, `eval/*`, `operators/dispatch.rs` deliberately left at `debug!` — their events fire once per operator invocation, not per row, so they're the right level for targeted debug sessions.
- [x] 1.3 `#[tracing::instrument(skip_all, level = "debug", fields(cypher = %query.cypher))]` added to `Executor::execute` — under `RUST_LOG=nexus_core=debug` callers now get one span per query with the Cypher source as a field, nested spans for CREATE / EXPAND, and no per-line clutter. `execute_expand` (expand.rs) got `fields(source_var, target_var, rel_var, optional, types = type_ids.len())` so the span carries the relationship shape; `execute_create_with_context` (create.rs) got `skip_all, level = "debug"` — the pattern argument is Debug but noisy, so structured fields aren't useful here.
- [x] 1.4 `nexus-server/src/main.rs` default `EnvFilter` tightened on both branches: verbose path now emits `"nexus_server=debug,nexus_core=debug,tower_http=debug,hnsw_rs=warn"`, quiet path `"nexus_server=error,nexus_core=warn,tower_http=error,hnsw_rs=warn"`. The `hnsw_rs=warn` floor is load-bearing — in the `fix/memory-leak-v1` session 95 % of production logs were `Hnsw max_nb_connection 16 … entering PointIndexation drop` noise from the KNN crate's `info!` firehose. A doc-comment on the filter block explains why so a future "clean up the defaults" pass doesn't accidentally unmuzzle it.
- [x] 1.5 Swept non-test code for `println!` leftovers in `nexus-core/src/` and `nexus-server/src/`. Four hits in `nexus-core/src/storage/graph_engine/bench.rs` lines 604 / 606 / 634 / 636, all inside `#[cfg(test)] mod tests { ... }` (line 483 opens the block), so they're legitimate test-only output — the task's scope was non-test code, which is clean.
