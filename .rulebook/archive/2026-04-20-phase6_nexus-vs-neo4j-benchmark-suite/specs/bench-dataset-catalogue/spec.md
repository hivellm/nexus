# Benchmark Dataset Catalogue Spec

## ADDED Requirements

### Requirement: Three Named Datasets

The system SHALL ship three datasets: `micro`, `social`, `vector`.
Each dataset SHALL be described by a `Dataset` struct containing
`id`, `node_count`, `rel_count`, `labels`, `rel_types`, and
`sha256_digest`.

#### Scenario: Dataset registry lists three entries
When `nexus-bench list-datasets` is executed
Then the output SHALL include `micro`, `social`, `vector` and
  nothing else

### Requirement: Deterministic Generation

The `micro` dataset SHALL be generated from a fixed seed (42) so
that identical invocations produce identical output.

#### Scenario: Reproducible micro dataset
Given two generator runs on the same host
When both produce the `micro` dataset
Then the resulting dumps SHALL have identical SHA-256 digests

### Requirement: SHA-256 Integrity Check

Before each bench run, the harness SHALL verify that every dataset
dump's SHA-256 matches the expected digest recorded in
`tests/data/bench/<name>.sha256`. A mismatch SHALL abort the run.

#### Scenario: Corrupted dataset rejected
Given a `micro` dump whose bytes have been flipped
When the harness starts
Then the run SHALL abort with `ERR_DATASET_CORRUPT(micro)`

### Requirement: Engine-Neutral Load

For each dataset the system SHALL provide a load primitive that
works against any `BenchClient`, not only one engine.

#### Scenario: Same dataset loads into both engines
Given the `social` dataset
When `load_dataset(social)` is called against `NexusEmbedded`
And against `Neo4jBolt`
Then both calls SHALL succeed
And `MATCH (n) RETURN count(n)` SHALL return the same count on both

### Requirement: `social` Dataset Is LDBC SNB sf=0.1

The `social` dataset SHALL be generated by `ldbc-snb-datagen` with
scale factor 0.1. The generated raw output SHALL be cached under
`tests/data/bench/social/` and regenerated only when the harness's
generator hash changes.

#### Scenario: Cache hit skips regeneration
Given a valid cached `social` dump
When the harness runs a second time
Then the generator SHALL not run again
And the cache hit SHALL be logged at INFO
