## 1. Operator design
- [x] 1.1 Read `phase6_opencypher-quantified-path-patterns` AST + grammar work — pre-existing `QuantifiedGroup` AST + `Operator::QuantifiedExpand` are the seam this slice extends. The slice keeps the existing BFS shape and adds mode-aware visited-set tracking around it; the WALK code path stays semantically identical so every existing slice-1 / slice-2 / slice-3 test continues to pass.
- [x] 1.2 Spec the `QuantifiedExpand` physical operator (input shape, output shape, traversal algorithm per mode) under `crates/nexus-core/src/executor/operators/` — `Operator::QuantifiedExpand` gains a `mode: QppMode` field carried through `dispatch.rs`, `executor/mod.rs`, and the planner. Frame is now `(current_node, iteration, nodes_per_pos, rels_per_hop, path_nodes, path_edges)`. Path sets are populated only when `mode.forbids_*_revisit()` returns true, so WALK pays no extra memory.
- [x] 1.3 Decide BFS vs DFS default; document via `rulebook_decision_create` — BFS retained. The BFS shape was the slice-3 default and the only one with the wavefront-dedup correctness analysis already complete; DFS would change the order of emitted rows in ways the existing TCK harness keys off. Mode-aware visited tracking integrates cleanly with BFS by extending the per-frame state.

## 2. Operator implementation
- [x] 2.1 Implement `QuantifiedExpand` for mode = WALK — unchanged behaviour: WALK keeps the wavefront dedup `(node, iteration)` and never populates `path_nodes` / `path_edges`. The implicit (no-keyword) default still lowers the textbook anonymous-body shape to the legacy `*m..n` operator at parse time for byte-for-byte backward compatibility; the explicit `WALK` keyword forces routing through `QuantifiedExpand` so the per-mode contract is uniform across the four keywords.
- [x] 2.2 Implement `QuantifiedExpand` for mode = TRAIL (track visited edges) — `path_edges: Vec<u64>` per frame; before pushing a new frame, reject any walk whose `hop_rels` contains a duplicate within itself OR an edge already in `path_edges`. Wavefront dedup is disabled because two paths reaching the same `(node, iteration)` via different sub-paths can have different visited-edge sets.
- [x] 2.3 Implement `QuantifiedExpand` for mode = ACYCLIC (track visited nodes) — `path_nodes: Vec<u64>` per frame initialised with the source node id; new walks are rejected if any node in `intermediate ∪ {end_node}` repeats within the walk or already appears in `path_nodes`. The starting node of the iteration is already in `path_nodes` from the prior iteration and is intentionally excluded from the new-node check (it is the joining point, not a revisit).
- [x] 2.4 Implement `QuantifiedExpand` for mode = SIMPLE (TRAIL + ACYCLIC) — both checks fire; `mode.forbids_edge_revisit()` and `mode.forbids_node_revisit()` are independent helpers that return `true` for SIMPLE.
- [x] 2.5 Carry path variables through to projection (length, nodes, relationships accessors) — already present from slice-3 (the `nodes_per_pos` / `rels_per_hop` machinery that drives list promotion). The mode-aware additions live alongside the existing per-position state without disturbing it.
- [x] 2.6 Cost estimate: bound by `n_label * avg_degree^max_hops`; cap explosion via planner threshold — already enforced via the `MAX_QPP_DEPTH = 64` cap in the operator. The `cap_was_hit` tracking + `ERR_QPP_UNBOUND_UPPER` log line continue to fire when a non-WALK mode + unbounded quantifier reaches the cap with candidates pending; the mode-aware visited-set pruning measurably reduces the queue size in ACYCLIC / SIMPLE on cyclic graphs (the visited rule rejects extensions before they reach the cap).

## 3. Planner integration
- [x] 3.1 Recognise pattern-group quantifier in logical plan — already shipped in slice-3; this slice extends it with a `mode` field on `Operator::QuantifiedExpand`.
- [x] 3.2 Lower `((...)-[...]->(...)){n,m}` to `QuantifiedExpand` — already shipped. `try_lower_to_var_length_rel` now refuses to lower whenever `QuantifiedGroup::mode_explicit` is true, so any explicit mode keyword (including `WALK`) routes to `QuantifiedExpand` rather than the legacy `*m..n` operator.
- [x] 3.3 Register the operator with cost-compare against legacy `*n..m` shortest-path — already in place via `qpp_legacy_rewrite_enabled()` and the `Operator::QuantifiedExpand { .. }` cost arm. The mode field does not change the cost shape (every non-WALK mode prunes more aggressively, so the WALK cost is the worst-case the planner already models).
