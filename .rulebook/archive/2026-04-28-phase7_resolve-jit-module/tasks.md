## 1. Decision
- [x] 1.1 Audited `crates/nexus-core/src/execution/jit/`: 1320 LOC across `mod.rs` + `codegen.rs` + `runtime.rs`, plus a 173-line `cranelift_jit.rs.disabled` shadow file, 12 placeholder markers under `Re-enable after core optimizations`. The module exported `JitRuntime` + `QueryHints` via `execution/mod.rs::pub use jit::{JitRuntime, QueryHints}` but a `grep -rn` across the workspace confirmed **zero production callers** anywhere (`nexus-server`, `nexus-cli`, `nexus-protocol`, `nexus-bench`, integration tests). The executor's only call site (`executor/mod.rs:92`) had been commented out for an unspecified period.
- [x] 1.2 Picked **Option B (delete)** and captured the rationale via `rulebook_decision_create` as ADR `delete-the-unused-jit-scaffold-rather-than-finish-the-cranelift-codegen` (id 8). Option A rejected because (a) no concrete near-term consumer asked for compiled queries, (b) the columnar fast-path real-world ratio is already ~1.13× per `PERFORMANCE_V1.md` so the gain a JIT would deliver is dominated by materialisation cost, (c) the planner's bigger leverage is cardinality propagation per `docs/analysis/nexus/02_architecture_assessment.md` §4.
