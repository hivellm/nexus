## 1. Implementation
- [x] 1.1 Inventory every call-site in `nexus-cli/src/client.rs` + `commands/*.rs` that hits an HTTP endpoint; map each to the RPC verb already served by `nexus-server/src/protocol/rpc/dispatch/*` (or mark as "HTTP-only for now" with a reason) — `query` → `CYPHER`, `ping` → `PING` (RPC-native), everything else stays HTTP-only for now (no `AUTH/USERS`, `STATS` admin verbs yet, export/import is REST-only). The auto-fallback keeps working so no command regresses.
- [x] 1.2 Add `nexus-protocol` as a dependency of `nexus-cli` and expose the RPC codec + types (done in `nexus-cli/Cargo.toml`; `rmp-serde` is pulled transitively via `nexus-protocol`).
- [x] 1.3 Create `nexus-cli/src/rpc_transport.rs` implementing the RPC client surface the CLI needs — picked a lightweight direct module (`RpcTransport { endpoint, credentials, Mutex<Option<BufReader<TcpStream>>>, AtomicU32 id }` with `async fn call(&self, cmd, args) -> Result<NexusValue>`) instead of a trait+two-impl scaffold so the CLI stays a single binary and commands keep using `NexusClient` unchanged.
- [x] 1.4 `RpcTransport::open` handles `HELLO 1` + optional `AUTH <api_key>` / `AUTH <user> <pass>` + request correlation (monotonic `AtomicU32` ids skipping `PUSH_ID = u32::MAX`).
- [x] 1.5 Existing HTTP code in `client.rs` stays as the fallback path; `NexusClient` now dispatches `query()` / `ping()` on the `Endpoint` scheme (RPC for `nexus://`, HTTP otherwise) and the other methods use the HTTP fallback against the sibling `15474` port.
- [x] 1.6 Replace the `http://localhost:3000` default in `nexus-cli/src/config.rs` with `nexus://127.0.0.1:15475`; `NexusClient::new` calls `Endpoint::default_local()` when no URL is supplied.
- [x] 1.7 Add `--transport rpc|http|https|auto|nexus` flag + `NEXUS_TRANSPORT` env var parsing in `nexus-cli/src/main.rs`; URL-scheme parser in `nexus-cli/src/endpoint.rs` accepts `nexus://`, `http://`, `https://`, and bare `host[:port]` (RPC default). `nexus-rpc://` and `nexus+rpc://` are explicitly rejected per the user directive to standardise on the single-token `nexus` scheme.
- [x] 1.8 Commands unchanged: they keep calling `NexusClient::{query, ping, stats, ...}` which now dispatches on the active transport. Commands without an RPC verb transparently use HTTP (no noisy fallback banner on every invocation — only the `--verbose` path prints the transport choice).
- [x] 1.9 `cargo +nightly fmt --all` + `cargo +nightly clippy -p nexus-cli --all-targets --all-features -- -D warnings` clean.
