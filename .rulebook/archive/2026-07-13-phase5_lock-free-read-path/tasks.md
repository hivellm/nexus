## 2. Read-only classification
- [x] 2.1 Added `routing::is_read_only(ast) -> bool` (`crates/nexus-server/src/api/cypher/routing.rs`) ŌĆö exhaustive per-clause match (no wildcard arm, so a future new `Clause` variant fails to compile until classified): write clauses (Create/Merge/Set/Delete/Remove/Foreach/LoadCsv) and all DDL/admin/user/API-key/transaction/SHOW commands are `false`; `CALL { ... }` and `PROFILE` recurse into the nested query; `EXPLAIN` is always `true` (never executes); `CALL procedure(...)` is `true` only against an explicit `READ_ONLY_PROCEDURES` allow-list (conservative default `false` for unknown/`apoc.*`/GDS-style procedures).
- [x] 2.2 37-case unit-test table in `routing.rs` (12 pre-existing `needs_engine_interception` cases + 25 new `is_read_only` cases): OPTIONAL MATCH, WITH-only, UNWIND-read, UNION-of-reads, CREATE/MERGE/SET/DELETE/REMOVE/FOREACH/LOAD CSV/BEGIN/SHOW DATABASES/CREATE INDEX (all `false`), EXPLAIN-of-write (`true`), EXPLAIN-of-read (`true`), PROFILE-of-read (`true`), PROFILE-of-write (`false`), CALL-subquery read/write, known read-only procedure, known write procedure, unknown procedure (apoc.*, gds.*) conservatively `false`.

## 3. Routing
- [x] 3.1 HTTP handler (`api/cypher/execute/handler.rs`): inside the existing `needs_engine_interception` branch, `routing::is_read_only(&ast)` gates a new sub-branch that takes a brief shared `server.engine.read().await` to clone `Engine::executor` and check the "default" session's `has_active_transaction()`; when not in an explicit transaction, executes the cloned `Executor` in `tokio::task::spawn_blocking` (mirrors the pre-existing bare-`RETURN` fallback's executor-clone/spawn_blocking/response-shaping/notifications shape) with no lock held during execution. In-transaction reads fall through unchanged to the pre-existing `engine.write().await` + `execute_cypher_with_params` path.
- [x] 3.2 RPC dispatch (`protocol/rpc/dispatch/cypher.rs::execute_query`): identical gate inside its `needs_engine_interception` branch ŌĆö same shared-read-lock executor clone + tx check, same `spawn_blocking` + `result_set_to_nexus` response shaping on the non-transactional path, same fallthrough to `engine.write().await` when an explicit transaction is open.
- [x] 3.3 Snapshot freshness: `Engine::executor` (distinct from `NexusServer::executor`, which is a permanently isolated `Executor::default()` used only by the pre-existing bare-literal fallback and unrelated to real graph data) is already replaced with a fresh snapshot by `Engine::refresh_executor()` before every commit/rollback/standalone-write's `engine.write().await` guard is released (pre-existing nexus-core mechanism, not modified). Because the lock-free path only clones `Engine::executor` after this branch's own `engine.read().await` is granted, the clone is guaranteed to postdate any write whose exclusive guard already released. Verified by `lock_free_read_path_test::lock_free_read_sees_immediately_preceding_committed_write` and parity-harness cases 17a/17b (BEGIN/CREATE/COMMIT and BEGIN/CREATE/ROLLBACK visibility, now exercised through the lock-free path since the post-tx MATCH is autocommit).
