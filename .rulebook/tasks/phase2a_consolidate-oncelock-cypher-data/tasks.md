## 1. Implementation
- [x] 1.1 Audited call sites: cypher/mod.rs had `EXECUTOR`, `_EXECUTOR_SHARED` (unused), `ENGINE`, `DATABASE_MANAGER`; cypher/execute.rs had three `ENGINE.get()` sites + one `EXECUTOR.get()`; data.rs had `CATALOG` (never initialised), `EXECUTOR` (never read), `ENGINE` (seven handler sites + four test sites). Every runtime consumer's target value was already owned by `NexusServer` (`server.engine`, `server.executor`, `server.database_manager`).
- [x] 1.2 Migrated `execute_cypher` to read `server.engine` / `server.executor` directly; the three `ENGINE.get()` / `EXECUTOR.get()` paths in execute.rs collapsed to `server.engine.write().await` / `server.executor.clone()`. The cache-ops helpers (`get_cache_stats`, `clear_cache`, `clean_cache`) now take `State<Arc<NexusServer>>`.
- [x] 1.3 Migrated every data.rs handler (`create_node`, `create_rel`, `update_node`, `delete_node`, `get_node_by_id`) to take `State<Arc<NexusServer>>`; the engine-not-initialised error branches collapse because the server always has an engine.
- [x] 1.4 Replaced `api::cypher::init_executor()` with a pure `build_executor()` helper that constructs an `Executor` with the intelligent query-cache config enabled. `main.rs` wraps the result in an `Arc` and hands it to `NexusServer::new`; `executor.set_database_manager(...)` moved into `main.rs` too.
- [x] 1.5 Deleted the `EXECUTOR`, `_EXECUTOR_SHARED`, `ENGINE`, `DATABASE_MANAGER` statics from cypher/mod.rs plus `init_executor` / `init_engine` / `init_database_manager` / `init_engine_with_executor` / `get_executor` / `get_database_manager`. Deleted the `CATALOG`, `EXECUTOR`, `ENGINE` statics from data.rs plus `init_catalog` / `init_executor` / `init_engine`. Removed the corresponding `api::cypher::init_*(...)` and `api::data::init_engine(...)` calls from `nexus-server/src/main.rs`. `api::data::init_engine` call in `tests/vectorizer_integration_test.rs` also dropped.
- [x] 1.6 Replaced the in-module test suite of `api::data::tests` with a parallel-safe set of four `#[tokio::test]` cases that build a fresh `Arc<NexusServer>` per test via a local `build_test_server()` helper (validation-only tests, a full create-and-get round-trip, and the isolation test from item 2.2).
- [x] 1.7 `cargo +nightly check -p nexus-server` clean; `cargo +nightly clippy -p nexus-server --all-targets --all-features -- -D warnings` clean.
